# contest-2026

## 프로젝트 개요
공공데이터 활용 공모전 A (마감 2026-04-12) 제출용 웹 서비스.
서비스명: **케어패스 (CarePass)** — 돌봄 가족을 위한 실시간 이동 지원 안내
피벗 옵션: **빈타임** (교통약자 API 실시간 불가 시)

## 기술 스택
- Next.js 15 App Router + TypeScript
- 카카오 지도 SDK (또는 네이버)
- SWR (실시간 데이터 갱신)
- Tailwind CSS
- 배포: Vercel

## 핵심 공공 API
- 교통약자 이동지원 현황 실시간 정보 (공공데이터포털)
- 전국 초정밀 버스 실시간 위치 정보 (공공데이터포털)
- 아이돌봄 서비스 제공기관 현황 (한국건강가정진흥원 / 공공데이터포털)

## 서비스 구조 (케어패스 기준)
사용자 위치 → 근처 아이돌봄 기관 목록 표시 →
기관 선택 → 교통약자 차량 현황 + 버스 도착 정보 동시 표시

## 피벗 조건 (빈타임)
교통약자 API가 실시간 차량 가용 데이터 미제공 시:
민원실 혼잡도 + 도서관 열람실 현황 + 버스 도착 기반 방문 타이밍 최적화 서비스로 전환

## Codex 전달 규칙

Codex에 작업을 넘길 때 프롬프트 최상단에 항상 포함:
- "최대 자원을 사용해 작업하라"
- 보고가 필요한 작업이면: "작업 완료 후 결과를 PT 형식으로 출력하라 (완료 내용 / 수정 파일 / 검증 결과 / 다음 추천 작업)"

## 금지 사항
- 턴바이턴 경로 최적화 (네비게이션 수준)
- 회원가입 / 로그인
- 교통약자 차량 직접 예약 연동
- 복수 경유지 최적화

## 7일 일정
D1: API 테스트 + 프로젝트 구조
D2: 기관 위치 데이터 + 지도 렌더링
D3: 핵심 UI (기관 선택 → 이동 수단 안내)
D4: 실시간 갱신 + 폴백 처리
D5: 반응형 모바일 UI
D6: Vercel 배포 + 데모 시나리오 확인
D7: 제출 자료 준비
